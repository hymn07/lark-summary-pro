# 07 — 部署运维规范

> 作用：让项目能稳定部署到生产环境，出问题能快速排查

---

## 一、硬性规则（MUST）

> ⚠️ 规则冲突提醒：如果以下规则与你的实际需求有任何冲突，AI 必须主动指出并建议替代方案（参见 00-AI协作原则.md）。

### 1.1 用 Docker Compose 编排

```yaml
services:
  postgres:     # 数据库（volume 持久化）
  web:          # Next.js standalone 应用
  nginx:        # 反向代理 + SSL
```

### 1.2 环境变量通过 .env 注入

```yaml
# ✅ 正确
services:
  web:
    env_file: .env

# ❌ 错误：硬编码在 docker-compose.yml 里
```

### 1.3 Migration 独立执行

```yaml
services:
  migrate:
    image: your-app-migrate:latest
    depends_on:
      postgres:
        condition: service_healthy
    # 跑完就退出，是一次性容器
```

### 1.4 数据必须持久化

```yaml
volumes:
  postgres_data:

services:
  postgres:
    volumes:
      - postgres_data:/var/lib/postgresql/data  # 容器删除数据不丢
```

### 1.5 健康检查

```yaml
services:
  postgres:
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s

  web:
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/api/health"]
      interval: 30s
```

## 二、强烈建议（SHOULD）

### 2.1 最小化 Docker 镜像

```dockerfile
FROM node:22-alpine
COPY .next/standalone ./
COPY .next/static ./.next/static
COPY public ./public
EXPOSE 3000
CMD ["node", "server.js"]
```

### 2.2 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://web:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 2.3 CI/CD 流程

```
Git push → CI build → push to registry → SSH 到服务器 → docker compose pull && up -d
```

### 2.4 安全头配置

在 `next.config.ts` 中配置：
```
Strict-Transport-Security
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: strict-origin-when-cross-origin
Content-Security-Policy
```

## 三、排查问题标准流程

```
1. docker ps                              → 容器在不在？
2. docker logs web --tail 100             → 有什么错误？
3. curl -v http://localhost:3000/api/health → API 通不通？
4. docker exec postgres psql -U user -d db  → 数据库连得上吗？
5. docker exec web env | grep DATABASE_URL   → 环境变量对不对？
```

## 四、部署常遇到的坑

1. **环境变量没配** → 应用连不上数据库 → 检查 `.env` 是否被 Docker 加载
2. **Migration 没跑** → 表不存在 → 确认 migrate 容器执行了
3. **端口没开** → 防火墙/安全组阻止 → 检查云服务商的安全组规则
4. **pgvector 等扩展** → PostgreSQL 镜像默认不带 pgvector → 用 `pgvector/pgvector:pg16` 镜像
5. **Docker Hub 网络不通** → 在国内服务器上需要配置镜像加速器或使用代理
6. **文件权限** → Docker volume 的所属用户与容器内用户不一致导致无法写入
